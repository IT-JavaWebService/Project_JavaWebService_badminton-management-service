HƯỚNG DẪN TEST API BẰNG POSTMAN (DỰ ÁN ĐẶT SÂN CẦU LÔNG - NGÀY 1 & NGÀY 2)
========================================================================

CÁC BƯỚC KHỞI ĐỘNG SERVER
-------------------------
1. Mở terminal tại thư mục D:\JavaRikkei\JavaWebService\Project\badminton_booking
2. Chạy lệnh: .\gradlew bootRun
3. Mở Postman để bắt đầu test các endpoint dưới đây.

---

I. NHÓM PUBLIC ENDPOINTS (/api/v1/auth/**)
------------------------------------------

1. Đăng ký tài khoản (FR-04)
   - Method: POST
   - URL: http://localhost:8080/api/v1/auth/register
   - Body (JSON):
     {
         "username": "customer1",
         "email": "customer1@gmail.com",
         "password": "password123",
         "role": "CUSTOMER"
     }
   *(Lưu ý: Có thể tạo thêm tài khoản với role là ADMIN hoặc MANAGER để test các quyền hạn tương ứng)*

2. Đăng nhập / Lấy JWT Token (FR-01)
   - Method: POST
   - URL: http://localhost:8080/api/v1/auth/login
   - Body (JSON):
     {
         "email": "customer1@gmail.com",
         "password": "password123"
     }
   - Response thành công sẽ trả về:
     {
         "success": true,
         "message": "Login successful",
         "data": {
             "accessToken": "eyJhbGciOi...",
             "refreshToken": "eyJhbGciOi...",
             "tokenType": "Bearer"
         }
     }
   *(Lưu ý QUAN TRỌNG: Hãy copy giá trị accessToken này để sử dụng làm Bearer Token cho các request có bảo mật phía dưới)*

3. Làm mới Token / Refresh Token (FR-02)
   - Method: POST
   - URL: http://localhost:8080/api/v1/auth/refresh
   - Body (JSON):
     {
         "refreshToken": "Copy_Giá_Trị_refreshToken_Ở_Bước_Login"
     }

4. Quên mật khẩu (FR-10)
   - Method: POST
   - URL: http://localhost:8080/api/v1/auth/forgot-password
   - Body (JSON):
     {
         "email": "customer1@gmail.com"
     }
   - Response sẽ trả về một UUID resetToken giả lập. Hãy copy token này.

5. Khôi phục mật khẩu mới (FR-10)
   - Method: POST
   - URL: http://localhost:8080/api/v1/auth/reset-password
   - Body (JSON):
     {
         "resetToken": "Nhập_Reset_Token_Nhận_Được_Ở_Bước_Forgot_Password",
         "newPassword": "newpassword123"
     }

---

II. NHÓM ENDPOINT CẦN XÁC THỰC (AUTHENTICATED ENDPOINTS)
--------------------------------------------------------
* Cách thiết lập Auth trong Postman cho các request dưới đây:
  - Chọn tab "Authorization".
  - Chọn Type: "Bearer Token".
  - Dán đoạn "accessToken" nhận được khi đăng nhập thành công vào trường "Token".

1. Đăng xuất (FR-03)
   - Method: POST
   - URL: http://localhost:8080/api/v1/auth/logout
   - Authorization: Bearer Token (Sử dụng accessToken hiện tại)

2. Đổi mật khẩu (FR-10)
   - Method: PUT
   - URL: http://localhost:8080/api/v1/auth/change-password
   - Authorization: Bearer Token (Sử dụng accessToken hiện tại)
   - Body (JSON):
     {
         "oldPassword": "password123",
         "newPassword": "newpassword123"
     }

---

III. QUYỀN CUSTOMER (/api/v1/customer/**)
-----------------------------------------
* Cần đăng nhập bằng tài khoản có role là CUSTOMER và cấu hình Bearer Token.

1. Đặt lịch sân cầu lông (FR-06)
   - Method: POST
   - URL: http://localhost:8080/api/v1/customer/bookings
   - Body (JSON):
     {
         "courtId": 1,
         "bookingDate": "2026-07-01",
         "timeSlotId": 1
     }
   *(Trước khi test, hãy chắc chắn trong database đã có dữ liệu mẫu cho courts và time_slots)*

2. Xem lịch sử đặt sân (FR-07)
   - Method: GET
   - URL: http://localhost:8080/api/v1/customer/bookings

---

IV. QUYỀN MANAGER & ADMIN (/api/v1/manager/**)
---------------------------------------------
* Đăng nhập bằng tài khoản có role là MANAGER hoặc ADMIN.

1. Phê duyệt lịch đặt sân (FR-08)
   - Method: PUT
   - URL: http://localhost:8080/api/v1/manager/bookings/1/approve
   *(Duyệt booking có id = 1 chuyển từ PENDING sang CONFIRMED)*

2. Từ chối lịch đặt sân (FR-08)
   - Method: PUT
   - URL: http://localhost:8080/api/v1/manager/bookings/1/reject
   *(Từ chối booking có id = 1 chuyển từ PENDING sang CANCELLED)*

3. Upload ảnh sân lên Cloudinary (FR-09)
   - Method: POST
   - URL: http://localhost:8080/api/v1/manager/courts/1/images
   - Body: Chọn kiểu "form-data" (không chọn JSON)
     - Key: "file" (chọn kiểu File thay vì Text)
     - Value: Chọn 1 ảnh có định dạng JPG hoặc PNG kích thước < 10MB từ máy tính của bạn.

---

V. QUYỀN ADMIN (/api/v1/admin/**)
---------------------------------
* Đăng nhập bằng tài khoản ADMIN.

1. Xem danh sách người dùng phân trang & tìm kiếm (FR-05)
   - Method: GET
   - URL: http://localhost:8080/api/v1/admin/users?page=0&size=10&search=customer

2. Xem chi tiết người dùng (FR-05)
   - Method: GET
   - URL: http://localhost:8080/api/v1/admin/users/1

3. Cập nhật thông tin người dùng (FR-05)
   - Method: PUT
   - URL: http://localhost:8080/api/v1/admin/users/1
   - Body (JSON):
     {
         "username": "customer1_updated",
         "email": "customer1@gmail.com",
         "role": "CUSTOMER",
         "active": true
     }

4. Xóa người dùng (FR-05)
   - Method: DELETE
   - URL: http://localhost:8080/api/v1/admin/users/1

VI. TÍNH NĂNG NÂNG CAO - NGÀY 3 (AF1 & AF3)
--------------------------------------------

1. Kiểm chứng Redis Token Blacklist (AF3):
   - Bước 1: Gọi API Đăng nhập (POST /api/v1/auth/login) để lấy accessToken.
   - Bước 2: Gọi API Đăng xuất (POST /api/v1/auth/logout) kèm Bearer Token vừa nhận. Lúc này Token sẽ được lưu vào Redis và tự động hết hạn theo TTL (Time-To-Live) của JWT.
   - Bước 3: Thử dùng lại accessToken đó để truy cập vào endpoint cần xác thực (VD: GET /api/v1/customer/bookings).
   - Kết quả mong muốn: Phản hồi trả về HTTP 403 Forbidden cùng body chứa thông báo:
     {
         "status": 403,
         "error": "Forbidden",
         "message": "Token is blacklisted",
         ...
     }

2. Kiểm tra AOP Performance & Audit Logging (AF1):
   - Bước 1: Thực hiện Đặt lịch sân thành công (POST /api/v1/customer/bookings).
     Kiểm tra log console của ứng dụng Spring Boot, kết quả mong muốn xuất hiện các dòng log dạng:
     [PERF] Method: createBooking | Duration: 23ms
     [AUDIT - SUCCESS] Customer customer1@gmail.com đặt thành công Sân Sân số 1, ngày 2026-07-10, khung giờ 08:00 - 09:00
   - Bước 2: Thực hiện Đặt lịch sân trùng giờ hoặc trùng ngày để cố tình gây lỗi.
     Kiểm tra log console của ứng dụng Spring Boot, kết quả mong muốn xuất hiện dòng log dạng:
     [AUDIT - FAILED] Customer customer1@gmail.com cố đặt Sân 1 nhưng thất bại. Lỗi: This time slot is already booked on the selected date
